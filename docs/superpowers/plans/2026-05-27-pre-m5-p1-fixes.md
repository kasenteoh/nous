## Task 1: Gate CIK backfill on `discovered_via`

**Problem:** `upsert_company` backfills CIK onto any matching `normalized_name` row regardless of source. When a VC-discovered "Acme" row exists and SEC indexes a different "Acme" with a CIK, they silently merge into one company.

**Fix:** Only accept the normalized-name match when the existing row is `discovered_via='form_d'`. Otherwise, fall through to INSERT — accepting an occasional duplicate as the cost of preventing a silent cross-entity merge.

**Files:**
- Modify: `pipeline/src/nous/db/upsert.py:106-113`
- Test: `pipeline/tests/test_upsert.py`

- [ ] **Step 1: Write the failing test**

async def test_upsert_company_does_not_hijack_non_form_d_row(
    db: AsyncSession,
) -> None:
    """Form D must NOT backfill CIK onto a row discovered by another source.

    Scenario: a VC-portfolio row exists for "Acme" (no CIK, discovered_via=
    'vc_portfolio'). SEC Form D arrives for a *different* "Acme" with a real
    CIK. Pre-fix, the CIK was backfilled onto the VC row, silently merging
    two distinct entities. After the fix, Form D inserts a new row.
    """
    from nous.util.slugify import normalize_name, slugify

    vc_row = Company(
        cik=None,
        name="Acme",
        slug=slugify("Acme"),
        normalized_name=normalize_name("Acme"),
        hq_country="US",
        discovered_via="vc_portfolio",
    )
    db.add(vc_row)
    await db.flush()
    vc_id = vc_row.id

    form_d = _make_form_d(
        cik="0009999001",
        entity_name="Acme",  # same normalized name
        accession_number="0009999001-26-000001",
    )
    company, created = await upsert_company(db, form_d)

    # A new row was inserted; the VC row is left alone.
    assert created is True
    assert company.id != vc_id
    assert company.cik == "0009999001"
    assert company.discovered_via == "form_d"

    # Verify VC row is untouched.
    refetched = await db.get(Company, vc_id)
    assert refetched is not None
    assert refetched.cik is None
    assert refetched.discovered_via == "vc_portfolio"
```

    # -- Step 1b: if CIK lookup missed, check by normalized name --
    # Only accept a name match when both rows are Form-D-sourced (the
    # original use case: an earlier no-CIK filing matched by name, current
    # filing supplies the CIK). For non-Form-D rows we fall through to
    # INSERT — accepting a possible duplicate is safer than hijacking a
    # VC/news/TC row that may represent a different real-world company.
    if company is None:
        candidate = await _find_by_normalized_name(session, norm)
        if candidate is not None:
            if not cik:
                # Without an incoming CIK, this is a re-ingest of the same
                # no-CIK Form D filing — match regardless of source.
                company = candidate
            elif not candidate.cik and candidate.discovered_via == "form_d":
                # Backfill CIK only onto a prior Form D row that lacked one.
                company = candidate
                company.cik = cik  # backfill
```

## Task 2: `funding_rounds.filing_id` → `ON DELETE SET NULL`

**Problem:** The FK has no `ondelete` clause, so it defaults to `NO ACTION` (deferred RESTRICT). Deleting a Filing directly is blocked even though the relationship is informational.

**Fix:** Set `ondelete="SET NULL"`. A FundingRound's news attribution stands on its own; losing the optional filing link is acceptable.

**Files:**
- Modify: `pipeline/src/nous/db/models.py` (FundingRound.filing_id)
- Create: `pipeline/alembic/versions/0006_funding_rounds_filing_id_set_null.py`

## Task 3: `extraction_confidence` CHECK constraint

**Problem:** Column is `String, nullable=True`. Application code at `upsert.py:370` uses `_CONFIDENCE_RANK.get(new, -1)`; a typo (`"medum"`) ranks as `-1` and silently downgrades the row. Pydantic catches typos at the LLM boundary but the DB is permissive.

**Fix:** Add a CHECK constraint allowing only `'low'`, `'medium'`, `'high'`, or NULL.

**Files:**
- Modify: `pipeline/src/nous/db/models.py` (FundingRound `__table_args__`)

## Task 5: `scrape_homepages` — back off on dead URLs

**Problem:** The eligibility query selects companies with zero `raw_pages` or stale pages. On fetch failure (robots block, 404, network error), nothing is persisted, so the same company is selected every weekly run. Wasted bandwidth and rate budget; violates the spirit of "1 req/s per domain".

**Fix:** Add `companies.last_scrape_attempt_at TIMESTAMPTZ`. Update on every attempt (success or failure). Eligibility excludes companies that were attempted recently but produced no pages.

**Files:**
- Create: `pipeline/alembic/versions/0008_companies_last_scrape_attempt_at.py`
- Modify: `pipeline/src/nous/db/models.py` (add column)
- Modify: `pipeline/src/nous/pipeline/scrape_homepages.py` (eligibility + write)
- Test: `pipeline/tests/test_scrape_homepages.py`

**Back-off policy (kept simple — refine in a future task if needed):**
- Successful scrape: rely on existing `raw_pages.fetched_at` and the `refetch_after_days` window (90 days default). No new gating needed.
- Failed attempt (zero pages produced this run): back off for `failure_backoff_days` (default 30 days) before re-attempting.
- `last_scrape_attempt_at` is set unconditionally at the end of every per-company iteration.

    failure_cutoff = datetime.now(tz=UTC) - timedelta(days=failure_backoff_days)

    # Find companies with a website that have no recent raw_pages.
    latest_fetch_subq = (
        select(
            RawPage.company_id,
            func.max(RawPage.fetched_at).label("latest_fetched_at"),
        )
        .group_by(RawPage.company_id)
        .subquery()
    )

    stmt = (
        select(Company)
        .outerjoin(
            latest_fetch_subq,
            Company.id == latest_fetch_subq.c.company_id,
        )
        .where(Company.website.is_not(None))
        .where(
            (latest_fetch_subq.c.latest_fetched_at.is_(None))
            | (latest_fetch_subq.c.latest_fetched_at < cutoff)
        )
        # Failure back-off: if a company has zero pages on record and was
        # attempted recently, suppress retry until the back-off elapses.
        # Companies WITH at least one page stay eligible whenever the
        # refetch_after_days predicate above fires.
        .where(
            (latest_fetch_subq.c.latest_fetched_at.is_not(None))
            | (Company.last_scrape_attempt_at.is_(None))
            | (Company.last_scrape_attempt_at < failure_cutoff)
        )
    )
    if limit is not None:
        stmt = stmt.limit(limit)

            homepage_content, used_browser = await _resolve_content_with_fallback(
                homepage, browser_client
            )
            if used_browser:
                summary.pages_via_browser_fallback += 1
            await _persist_fetched_page(

            )
            summary.pages_fetched += 1

            # Step 2: discover relevant subpages.
            discovered = _extract_relevant_links(
                homepage_content,
                base_url=homepage.url,
                max_links=max_extra_pages,
            )

            # Step 3: fetch each discovered link.
            for sub_url in discovered:

                if sub == "robots":

                    continue
                if sub is None:

                    sub, browser_client
                )
                if sub_used_browser:
                    summary.pages_via_browser_fallback += 1
                await _persist_fetched_page(
                    session, company.id, url=sub.url, content=sub_content
                )
                summary.pages_fetched += 1
        finally:
            company.last_scrape_attempt_at = datetime.now(tz=UTC)
            session.add(company)
            await session.commit()
